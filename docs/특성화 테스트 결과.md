표)과 함께 읽으면 "표에는 있지만 테스트로 고정되지 않은 규칙"이 무엇인지 바로 알 수 있다.

## 측정 방법

`pom.xml`에는 JaCoCo가 설정되어 있지 않아, 커맨드라인에서 임시로 실행해 측정했다(pom.xml 변경 없음).

```
./mvnw.cmd org.jacoco:jacoco-maven-plugin:0.8.12:prepare-agent test \
           org.jacoco:jacoco-maven-plugin:0.8.12:report \
           -Dtest=ApprovalServiceProcessApprovalCharacterizationTest
```

## 패턴으로 본 공백

- **"우선순위 자동 상향" 규칙(#3)이 전혀 검증되지 않음** — `docs/4-13`에 명시한 상신의 핵심 부가 효과인데,
  6개 테스트 모두 금액을 임계값 미만으로 설정해서 피해갔다. 가장 먼저 채워야 할 공백이다.
- **반려(REJECT)가 승인(APPROVE)만큼 검증되지 않음** — 승인은 "성공+권한없음" 2가지를 테스트했지만
  반려는 "성공"만 있고 권한없음(#9)·상태불일치(#7)·잘못된 결재자(#8)가 비어 있다. 승인/반려는
  `docs/4-13`에서 이미 "완전히 동일한 전제조건"이라고 밝혔으므로, 반려 쪽에 승인과 대칭되는 테스트를
  추가하면 이 공백 대부분이 해소된다.
- **"조회는 되지만 지정된 사람이 아님"(#5, #8, #13) 이 세 곳 모두 비어 있음** — 현재 6개 테스트는
  권한 검사(role) 실패만 다루고, "아예 지정되지 않은 사람이 접근"하는 경우를 다루지 않는다.
- **취소(CANCEL)의 상태 조건(#12)이 4분의 1만 커버** — 임시저장 상태 취소만 확인했고, 상신 상태 취소·
  이미 종결된 상태에서의 취소 시도가 비어 있다.
- **연쇄 관계 관리 데이터 결측(#4, #6, #10)** — 결재자/기안자 계정이 사라진 상태는 실제 운영에서
  드물게 발생할 수 있는 데이터 정합성 문제이며, 우선순위는 낮지만 "메일 발송 스킵"이라는 관찰 가능한
  동작이라 특성화 대상이다.
- **정의되지 않은 action 값(#11)** — API 계약상 1/2/3/9 외의 값이 들어왔을 때 "완전히 무시된다"는
  것도 관찰 가능한 동작이므로 특성화할 가치가 있다.
